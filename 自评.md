## 最得意的一个技术决策

**无 API Key 也能完整验证全流程**：embedding/LLM 在 `key=replace_me` 时走确定性 mock（mock 向量、mock 工具选择），检索用字符重叠近似。这样不依赖真实密钥即可验证上传→索引→检索→引用→拒答→工具调用→多轮→IM 兜底的完整链路，大幅降低开发与面试演示门槛。

## 一个被迫的妥协

mock 模式下 LLM 回答是"工具结果摘要"而非自然语言（真实语义需配置 API Key）。工具选择的 mock 启发式是测试替身，真实模式由 function calling 自主决定（无 if-else）。
