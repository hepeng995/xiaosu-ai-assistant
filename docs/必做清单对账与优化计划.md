## 一、背景与目的

本文档对笔试题「必做清单」逐条核对当前实现，找出**尚未完成或存在短板**的项，并给出**分波次的优化任务清单**。核对方式为实读后端 `apps/api/app` 与前端 `apps/web/app` 的源码、测试与迁移，而非依赖文档自述。

**核心结论**：

- **必做清单（6 大功能 / 30+ 子项）：仅 1 项硬缺口** —— 功能 5「设置 - 切换模型」未实现（设置页纯只读，`Setting` 表定义了却未被任何代码使用）。
- **加分项短板**：Token 成本链路「断头」（`estimated_cost` 字段存在但从未计算，前端也未展示）；Langfuse 仅配置位（noop）、无 Evals、无多模型适配、前端未用 shadcn。

优化全程守住红线：**未配置即降级 mock**、**单文件 ≤ 500 / 单目录 ≤ 8**、**工具选择禁硬编码**、**RAG 必带引用无依据必拒答**、**密钥不入库不打印**。
