## 一、设计目标

招聘题第七章定义了 9 道验收题（7.1 基础问答 / 7.2 工具调用 / 7.3 多轮 / 7.4 拒答）。其中 **RAG 知识库只需精准拿下三类场景**：

1. 🟢 **带引用命中**：Q1 年假、Q2 报销材料、Q3 入职流程 —— 答案必须来自知识库并附原文出处
2. 🔴 **干净拒答**：Q8 CEO 住址、Q9 2030 销售目标 —— 库中无依据时明确拒答，禁止编造
3. ⚪ **非 RAG 场景**：Q4–Q7 走 Tool Calling（员工/订单/考勤/时间 API），不经过知识库

> 核心原则：**RAG 不追求大而全，而追求「该命中的稳命中、该拒答的干净拒答」**。重心放在文档去重、标题结构化、阈值校准，而非堆砌文档或过度工程。

## 四、RAG 数据库设计

### 4.1 文档清单（5 篇，去重 + 格式覆盖）

| 文档 | 格式 | 职责 | 覆盖验收 | 设计要点 |
|------|------|------|---------|---------|
| **员工手册.md** | MD | 公司制度总纲（年假/加班/打卡/报销/出差） | Q1/Q2 | 权威源，报销部分写全 |
| **报销制度细则.docx** | DOCX | 报销深度细则（发票类型/限额/特殊场景） | Q2 加分 | **删除与手册重复的概览**，只留细则增量（兼测 DOCX） |
| **新人入职指南.md** | MD | 入职流程/设备/试用期 | Q3 | 已完善 |
| **FAQ.md**（升级自 .txt） | MD | 高频问答 | 多场景兜底 | **加 `## 问题` 标题**，让每个 Q 有 heading_path |
| **差旅与报销标准.pdf** | PDF | 差旅标准（交通/住宿限额） | 加分 | 测 PDF + 出差场景 |

**去重原则**：同一事实**唯一权威源**（手册管总纲，docx/pdf 管细则增量），避免多文件重复同一句话。

### 4.2 分块策略

| 参数 | 值 | 理由 |
|------|-----|------|
| `RAG_CHUNK_SIZE` | 800 | 中文制度段落多在 100–300 字，800 留足上下文又不过长 |
| `RAG_CHUNK_OVERLAP` | 120 | 跨块保留语义连续性 |
| **标题结构保留** | 必须 | 每个分块带完整 `heading_path`（如「员工手册>考勤制度>年假规则」），支撑招聘题「来源定位/高亮原文」 |
| **分块粒度** | 按标题语义段落 | 避免跨主题切碎，保证一个分块讲清一件事 |

### 4.3 检索策略

| 策略 | 配置 | 说明 |
|------|------|------|
| **向量检索** | pgvector 余弦 `1-(embedding <=> query)` | 主检索方式 |
| **阈值** | `RAG_SCORE_THRESHOLD = 0.72` | 与笔试题/开发规范一致；低分命中应通过查询改写或文档补充解决 |
| **top_k** | `RAG_TOP_K = 5` | 兼顾召回与噪声 |
| **query 改写** | LLM 自主改写 | Agent 调 `search_knowledge_base` 时 LLM 会改写查询（如「我能够休几天年假」→「年假天数 年休假制度」），无需额外开发 |
| **Embedding** | text-embedding-v4，dimensions=1536 | 与 `document_chunks.embedding VECTOR(1536)` 严格一致 |
| **（可选·加分）混合检索** | BM25 + 向量 | 提升中文精确词命中。当前已过验收，非必须 |

### 4.4 拒答边界设计（显式化）

| 边界类型 | 处理方式 | 保障层级 |
|---------|---------|---------|
| **隐私类**（CEO住址、个人薪资） | 库中**绝不收录** + 阈值拦截 + LLM 相关性判断 | 三重保险 |
| **未来/外部数据**（2030目标、行业数据） | 库中不收录 + 拒答 | 阈值 + LLM |
| **知识库无关**（闲聊、超纲） | 阈值拦截 + LLM 判断 | 双保险 |

> 拒答判定（`chat_service`）：仅当「调用的工具**全是**知识库检索且无引用命中」才判拒答；只要调过任何外部业务工具就采用模型回答。敏感词命中走 `REFUSAL_PRIVACY`。
